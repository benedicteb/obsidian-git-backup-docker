# Session 019 — Automated obsidian-headless Update Checker

## Date

2026-03-13

## Summary

Added a daily GitHub Actions workflow that checks npm for new versions
of obsidian-headless and opens a PR with the Dockerfile version bump,
upstream changelog, dependency diff, and an Obsidian-specific review
checklist.

## What was done

1. **Created `.github/workflows/check-headless-update.yml`** — a daily
   cron workflow (08:00 UTC) that:
   - Queries the npm registry for `dist-tags.latest`
   - Compares against the pinned version in the Dockerfile
   - Fetches the upstream CHANGELOG.md from GitHub (master branch)
   - Extracts changelog entries between the current and latest versions
   - Computes a dependency diff (added/removed/changed npm deps)
   - Builds commit comparison links from `gitHead` SHAs in npm metadata
   - Opens a PR with `dependencies` and `automated` labels
   - Deduplicates: skips if a PR already exists for the same version

2. **Addressed subagent reviews** — all three reviewers identified
   issues that were fixed:
   - **linux-expert**: Fixed shell injection risk (dep JSON via temp
     files instead of `${{ }}`), fixed `${VAR:0:7}` bashism (use `cut`),
     fixed `grep -oP` portability (use `grep -o` + `sed`), escaped sed
     regex dots, reordered PR body construction before branch push,
     added semver validation, fixed `awk` changelog extractor
   - **ux-designer**: Added `> [!WARNING]` callout, improved checklist
     with concrete actions, added upstream issues link, added PR labels,
     added failure documentation to header, added post-merge verification
     step
   - **obsidian-expert**: Added sync protocol compatibility check,
     `ob sync-config` API surface check (ADR-0017 dependency), E2EE
     credential format check, reassurance that failure mode is "sync
     stops" not data loss

3. **Created ADR-0019** documenting the decision, alternatives, and
   consequences.

## Decisions

- **ADR-0019**: Use a custom GitHub Actions workflow to check for
  obsidian-headless updates daily, rather than Dependabot/Renovate
  (which can't parse Dockerfile RUN commands) or manual monitoring.

## Open questions

- The awk changelog extractor uses string-based version comparison.
  If obsidian-headless reaches `0.10.x`, the comparison will break.
  This is documented in the workflow and ADR but not yet fixed.

- Bot-created PRs accumulate if not reviewed during a burst of upstream
  releases. Each version gets its own PR. Stale PRs must be closed
  manually.
