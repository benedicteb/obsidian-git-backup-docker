# Session 004 — Move SSH keys from /config/.ssh/ to /config/

**Date:** 2026-03-02

## Summary

Moved SSH key storage from the hidden `/config/.ssh/` subdirectory to
directly under `/config/`. This was requested because having a `.ssh`
dotfile directory inside a Docker bind mount is awkward — it's invisible
in `ls` and Finder by default, requires extra `mkdir -p` steps, and
provides no meaningful security benefit in a single-purpose container.

## Changes Made

### Core change: SSH path migration

- **init-config.sh**: Changed `SSH_DIR="/config/.ssh"` references to flat
  paths (`/config/id_ed25519`, `/config/ssh_config`, `/config/known_hosts`).
  Removed `.ssh` directory creation and `chmod 700` on it.
- **Dockerfile**: Removed `mkdir -p /config/.ssh` from the build-time
  directory setup.
- **README.md**: Updated all SSH path references, simplified bind-mount
  instructions (no more `mkdir -p .ssh` step), simplified volumes table.
- **docker-compose.yml**: Updated SSH path comment in the bind-mount
  example.

### Hardening (from subagent review feedback)

- **ssh_config is now conditional**: Only generated if absent. Users who
  provide a custom `ssh_config` (e.g., for ProxyJump, non-standard port)
  have it respected instead of silently overwritten on every restart.
- **chown failures are non-fatal**: On macOS bind mounts, `chown` to a
  different UID fails. The init script now warns instead of aborting
  (which would have broken the "bring your own key" workflow on macOS).
- **known_hosts is pre-created**: With correct ownership and permissions,
  avoiding umask-dependent behavior when SSH first connects.
- **ssh_config written with umask 077**: Eliminates the brief
  world-readable window between file creation and `chmod 600`.
- **SSH banner includes key path**: Helps users find the key file.

## Decisions

- **ADR-0003**: Documented the decision to store SSH files flat in
  `/config/` with rationale and trade-offs.
