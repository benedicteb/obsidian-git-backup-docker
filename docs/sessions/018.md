# Session 018 — Privacy-Friendly Docker Hub Pull Count Metrics

**Date:** 2026-03-11

## What was accomplished

Added a privacy-friendly metrics system to track Docker Hub image downloads
over time, without collecting any individual user data.

### Changes made

1. **`.github/workflows/docker-publish.yml`**
   - Added `paths-ignore: ['docs/metrics/**']` to prevent metrics commits
     from triggering Docker image builds.
   - Added `--invert-grep --author="github-actions[bot]"` to the changelog
     generator so bot commits don't appear in release notes.

2. **`.github/workflows/metrics.yml`** (new)
   - Daily cron workflow (06:00 UTC) that fetches Docker Hub's public
     `pull_count` API and appends a row to `docs/metrics/pulls.csv`.
   - Includes: numeric validation, CSV header integrity check,
     idempotency guard, concurrency group, explicit push target,
     `workflow_dispatch` for manual runs.

3. **`docs/metrics/pulls.csv`** (new)
   - CSV with `date,pull_count` header. The workflow appends one row per day.

4. **`README.md`**
   - Added Docker Pulls shields.io badge.
   - Added one-line mention of the CSV in the Publishing section.

5. **`docs/adr/0018-privacy-friendly-pull-count-metrics.md`** (new)
   - Documents the decision and trade-offs.

## Decisions made

- **ADR-0018**: Use Docker Hub's public `pull_count` API (a single
  anonymous aggregate counter) with a daily GitHub Actions scraper and a
  shields.io badge. No third-party analytics, no cookies, no user tracking.
