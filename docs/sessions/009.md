# Session 009 — Unraid Community Applications Template

**Date:** 2026-03-03

## Summary

Added an Unraid Community Applications (CA) template so Unraid users can
discover and install the container through the standard CA app-store
interface.

## What was done

1. **Researched Unraid CA template format** — Studied the XML schema,
   reviewed the selfhosters/unRAID-CA-templates repo, read the template
   authoring guide at selfhosters.net, and examined a production template
   (vaultwarden) as a reference.

2. **Created the template** (`unraid-template/obsidian-git-backup.xml`):
   - All required variables (auth token, vault name, git remote URL)
   - Volume mappings with Unraid-standard `/mnt/user/appdata/` paths
   - Optional variables (git author, branch, debounce, E2EE)
   - PUID/PGID defaulting to 99/100 (Unraid nobody/users)

3. **Created a placeholder icon** — SVG-based PNG (256x256) with a
   purple diamond and git-branch motif. Stored in
   `unraid-template/img/obsidian-git-backup.png`.

4. **Updated README** — Added "Unraid Installation" section with
   instructions for adding the template repository, configuring the
   container, and the SSH key setup step.

5. **Ran subagent reviews** — All three reviewers (linux-expert,
   ux-designer, obsidian-expert) provided detailed feedback.

6. **Addressed review feedback** — Fixed all HIGH, MEDIUM, and LOW
   severity issues identified by reviewers.

## Key decisions

- **Self-hosted template** (not submitting to selfhosters repo) — The
  selfhosters repo is winding down new requests and recommends self-hosted
  templates. See ADR-0007.

- **No `--restart unless-stopped`** — Removed per linux-expert review.
  Unraid manages container lifecycle through its own Autostart mechanism;
  Docker restart policies conflict with Unraid's UI.

- **E2EE password moved to `Display="always"`** — Users with encrypted
  vaults need to see this field immediately, not hunt through Advanced.

- **`/vault` marked `Required="true"`** — While data can be re-synced,
  the mount is effectively required for sane operation on Unraid.

### obsidian-expert

- Added E2EE plaintext-in-git warning

## Open questions

- **Icon quality**: The placeholder icon works but could be improved
  with a professional design. Replace `unraid-template/img/obsidian-git-backup.png`
  when a better icon is available.

- **selfhosters PR**: Could submit a PR to `selfhosters/unRAID-CA-templates`
  for wider CA discoverability, but this creates a second copy to maintain.

- **Template sync validation**: No automated check that the template XML
  stays in sync with Dockerfile ENV changes. Could add a CI check later.
