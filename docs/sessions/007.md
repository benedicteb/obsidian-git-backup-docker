# Session 007 — Add Timestamps to All Log Output

**Date:** 2026-03-02

## Summary

Added ISO 8601 UTC timestamps to all log output across the project by
creating a shared logging library and updating all scripts to use it.

## Changes Made

### New file: `rootfs/usr/local/lib/log-functions.sh`

Shared logging library providing `log()`, `log_error()`, and `log_banner()`
functions. Scripts set `LOG_TAG` before sourcing to identify themselves.

Output format: `2026-03-02T14:30:05Z [service-name] message`

### Updated scripts

- **`init-usermap.sh`** — Replaced inline `log()`/`log_error()` with sourced
  library.
- **`init-config.sh`** — Replaced inline functions, converted bare `echo`
  statements in SSH key banner to `log()`, removed blank `log_error ""` calls.
- **`git-watcher.sh`** — Replaced inline functions, moved logging setup before
  validation so early error messages also get timestamps.
- **`log-prefix.sh`** — Added timestamps to the FIFO reader loop output,
  matching the shared library format.

### Architecture decision

Recorded as ADR-0005: Shared Timestamped Logging Library.

## Subagent Reviews

All three reviewers were consulted. Key feedback addressed:

- **Linux expert**: Cache timestamp in `log_banner` (avoid divergence across
  lines), use UTC `Z` instead of `%z` (BusyBox produces `+0000`, not
  `+00:00`), add throughput/timing comments to `log-prefix.sh`.
- **UX designer**: Remove blank `log_error ""` calls (produce noisy
  metadata-only lines), timestamp banner blank lines for parseable format,
  document `docker logs --timestamps` double-timestamp trade-off.
- **Obsidian expert**: Add comment about reader-timing vs emission-timing in
  `log-prefix.sh`. Confirmed timestamps are valuable for correlating sync
  events with git-watcher commits.

## Decisions

- Use UTC with `Z` suffix instead of local timezone with `%z` — simpler,
  unambiguous, avoids BusyBox formatting quirks.
- Every log line is fully timestamped and tagged for parseable output —
  including banner decoration lines.
- Performance trade-off of one `date` fork per log line accepted for all
  current use cases.
