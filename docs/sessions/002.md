# Session 002 — Fix Auth Token Documentation

**Date:** 2026-03-02

## What was accomplished

Investigated the obsidian-headless source code (`cli.js`) to determine the
correct auth token storage mechanism, and fixed incorrect documentation in
both `README.md` and `.env.example`.

### The problem

The docs claimed the token was stored in `config.json` as a JSON object. This
was wrong — there is no `config.json`. The obsidian-headless source was
downloaded to `./obsidian-headless` and the minified `cli.js` was analyzed to
extract the actual behavior.

### Findings from source code analysis

The token is stored as a **plain text file** called `auth_token` (not JSON).
The config directory varies by platform:

| Platform | Token file path |
|---|---|
| Linux | `~/.config/obsidian-headless/auth_token` |
| macOS | `~/.obsidian-headless/auth_token` |
| Windows | `%USERPROFILE%\.obsidian-headless\auth_token` |

Key functions identified in the minified source:

- `J()` (getToken) — checks `OBSIDIAN_AUTH_TOKEN` env var first, then reads
  the `auth_token` file as plain text.
- `wt()` (saveToken) — writes the raw token string to `auth_token` with
  mode `0600`.
- `Oe()` (getTokenOrExit) — calls `J()`, exits with code 2 if no token found.

The `ob login` command also supports non-interactive use with `--email`,
`--password`, and `--mfa` flags.

### What was wrong in the docs

| | Old (wrong) | Correct |
|---|---|---|
| File name | `config.json` | `auth_token` |
| Format | JSON with `"token"` key | Plain text string |
| macOS path | `~/Library/Application Support/obsidian-headless/` | `~/.obsidian-headless/` |
| Windows path | `%APPDATA%\obsidian-headless\` | `%USERPROFILE%\.obsidian-headless\` |
| Linux path | `~/.config/obsidian-headless/` | `~/.config/obsidian-headless/` (was correct) |

### Files changed

- `README.md` — Corrected token file paths, format description, added `cat`
  examples for Linux and macOS.
- `.env.example` — Corrected token file paths and format description.
