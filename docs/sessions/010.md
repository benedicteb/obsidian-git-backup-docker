# Session 010: Unraid CA Template Review & Fixes

## Date

2026-03-03

## Summary

Conducted a comprehensive Unraid Community Applications template review
using the `@unraid-reviewer` subagent, then addressed all 16 findings
plus additional feedback from `@linux-expert`, `@ux-designer`, and
`@obsidian-expert` subagent reviews.

## What was accomplished

### Critical fixes

1. **Template discovery** — Moved `obsidian-git-backup.xml` from
   `unraid-template/` to the repository root. CA's template scraper only
   scans the root directory; the template was invisible to users.

2. **inotify watch limit** — Discovered that `--sysctl
   fs.inotify.max_user_watches=524288` in `<ExtraParams>` has no effect
   because it's not a namespaced sysctl. Replaced with host-side
   instructions in the `<Requires>` field.

3. **E2EE description** — Fixed factually incorrect claim that "E2EE
   protects the Obsidian Sync transport only." E2EE also protects data
   at rest on Obsidian's servers.

### Improvements from subagent reviews

4. **Field ordering** — Reordered to keep required fields contiguous:
   Auth Token → Vault Name → Git Remote URL → E2EE Password (optional).

5. **Auth token description** — Leads with security warning, includes
   Windows path, uses `find` command for cross-platform token discovery.

6. **PUID/PGID descriptions** — Replaced alarming "WARNING: Do not
   clear" with affirmative guidance ("Leave at 99 unless...").

7. **Vault Name** — Added vault ID (UUID) as alternative for duplicate
   vault names.

8. **Multi-vault guidance** — Added device limit warning and concurrent
   vault prohibition. Changed wording from "install template once per
   vault" to "add a second container."

9. **Config Storage** — Made "if lost" consequence concrete.

10. **Git Branch** — Added note that GitHub creates 'main' automatically.

11. **Vault Storage** — Improved cache drive advice to reference appdata
    share settings instead of suggesting path changes.

12. **SSH key post-install** — Added to Requires field.

13. **Outbound network** — Documented TCP 443 + TCP 22 requirements.

14. **Metadata** — Added `<Beta>true</Beta>`, `<Date>`, and `<Changes>`
    fields for CA update notifications.

### Documentation updates

15. **ADR-0007** — Updated to reflect template move to repo root.
16. **ADR-0009** — New ADR documenting the template move decision.
17. **README** — Updated Unraid section to reflect E2EE field placement.

## Icon URL decision

The `@unraid-reviewer` recommended keeping the `<Icon>` URL pinned to
`main` (not a version tag or SHA). Rationale:
- 100% of community template repos use `main`
- The icon is a placeholder that may be redesigned (auto-update is a
  feature, not a bug)
- Version-pinning would create CI complexity and a chicken-and-egg
  update cycle
- raw.githubusercontent.com has 5-minute CDN TTL, so broken icons
  self-heal quickly

## Decisions made

- **ADR-0009**: Template XML lives at repo root for CA discovery.
- Icon URL stays on `main` (community convention, no action needed).
- `--sysctl` ExtraParams removed; host-side inotify instructions added
  instead (not a namespaced sysctl).

## Open questions

1. **Device naming** — obsidian-headless registers all containers with
   the server hostname. A future `OBSIDIAN_GIT_DEVICE_NAME` env var
   could let users differentiate devices in Obsidian Sync settings.

2. **README vs template token paths** — The README uses explicit
   per-platform paths; the Unraid template uses a `find` command.
   Both are intentional — explicit paths suit the README's structured
   list format, while `find` suits the template's cramped single-line
   description. No unification needed.
