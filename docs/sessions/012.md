# Session 012: Fix Unraid CA Template Installation Instructions

**Date:** 2026-03-03

## Summary

Fixed incorrect Unraid installation instructions in the README. The
previous instructions directed users to a non-existent "Settings icon
(gear)" in the Apps tab to add a template repository URL. Research of
the current Unraid CA documentation confirmed that no such feature
exists in the Apps tab.

## Changes

1. **Replaced broken install method with CA private repository method**
   — Users now download the template XML to their flash drive at
   `/boot/config/plugins/community.applications/private/obsidian-git-backup/`.
   The template then appears in the Apps tab under a "Private" category.

2. **Replaced `wget` with `curl -fsSL`** — `wget` is not reliably
   available on older Unraid versions (pre-6.12). `curl` is available
   on all supported versions (6.9+). The `-f` flag prevents silent
   failure on HTTP errors.

3. **Added CA plugin prerequisite** — Explicit note that Community
   Applications must already be installed before running the commands.

4. **Fixed CA refresh instructions** — Must use CA's in-app refresh
   icon, not browser refresh (Ctrl+R). The previous "you may need to
   refresh" phrasing was misleading.

5. **Split terminal commands** — Separated `mkdir` and `curl` into
   separate code blocks for safer copy-paste in Unraid's web terminal.

6. **Made inotify go-file command idempotent** — Replaced raw
   `echo >>` append with `grep -q ... ||` guard to prevent duplicate
   lines on repeated runs.

7. **Fixed Docker tab UI labels** — "Log" → "Logs" (matching actual
   Unraid UI), "coloured square" → "name or icon" for context menu.

## Research Findings

- The Unraid CA "Template Repositories" field was historically on the
  **Docker** tab, not the Apps tab. Its availability varies across
  Unraid versions and it's not documented in the current official docs.
- As of June 2025, new template repositories should be submitted to
  the CA index via an Asana form (reviewed within 48 hours).
- The CA FAQ by Squid describes two methods for non-indexed templates:
  the "Old School" Docker tab field, and the "Preferred" private
  repository folder method.
- The official Unraid docs (docs.unraid.net) make no mention of
  Template Repositories at all — only the Asana submission workflow.

## Decisions

- **ADR-0011**: CA Private Repository install method — documents why
  this approach was chosen over the Docker tab field, direct `docker
  run`, or submitting to the official CA index.
