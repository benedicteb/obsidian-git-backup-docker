# Session 003 — PUID/PGID Privilege Separation

**Date:** 2026-03-02

## What was accomplished

Implemented the linuxserver.io PUID/PGID convention to fix "permission denied"
errors when using bind mounts on macOS. This was the most-reported issue in
the PoC and completely blocked macOS users from using local directories.

### Changes made

1. **Dockerfile**
   - Added `su-exec` and `shadow` packages for runtime UID/GID remapping
   - Created `obsidian` user/group (UID/GID 1000) replacing the base
     image's `node` user
   - Added `python3`, `make`, `g++` as build deps for `better-sqlite3`
     compilation (fixes pre-existing build failure)
   - Added PUID/PGID env vars with defaults (1000:1000)
   - Added `.dockerignore` to minimize build context

2. **init-usermap.sh** (new) — s6 oneshot that runs before init-config:
   - Validates PUID/PGID (numeric, non-zero)
   - Remaps obsidian user/group to match host UID/GID
   - Recursively fixes ownership of /config and /vault when needed
   - Actionable error messages with remediation hints

3. **init-config.sh** — Updated to delegate user-facing operations via
   `su-exec obsidian`:
   - SSH key generation, git clone/config, ob sync-setup
   - `.gitignore` management (still root, chown'd after)
   - Improved missing-env-var error to list vars as bullets

4. **s6 service definitions** — All run/finish scripts updated:
   - `with-contenv` shebang for env var injection
   - `su-exec obsidian` for privilege de-escalation
   - Fix finish script env var expansion (single→double quotes)

5. **Obsidian-specific improvements** (from obsidian-expert review):
   - Added `.obsidian/plugins/*/data.json` to vault .gitignore (prevents
     ghost commits from Dataview, Various Complements, etc.)
   - Added commented-out entries for appearance.json, hotkeys.json, etc.
   - Expanded inotifywait --exclude to match the .gitignore (prevents
     unnecessary watcher→debounce→"no changes" cycles)

6. **Documentation** — README, .env.example, docker-compose.yml all updated:
   - PUID/PGID section with macOS/Linux instructions
   - macOS callout in Quick Start
   - PUID/PGID note in Option B (bind mount)
   - Consistent wording: "Required on macOS with bind mounts"
   - Removed "Runs as root" from limitations
   - Updated architecture diagram with init-usermap

7. **docker-compose.yml env_file/environment fix** — Removed the
   `environment` block that was silently overriding PUID/PGID from the
   env file. See ADR-0002 "Lessons learned" for the full explanation.

## Key lessons learned

1. **Docker Compose `environment` overrides `env_file` — silently.**
   `${PUID:-1000}` in the `environment` block resolves from the host shell,
   not from the env file. If the user only sets PUID in their env file, the
   `environment` block defaults to 1000 and overrides the env file value.
   Fix: never duplicate a variable in both `env_file` and `environment`.
   Use Dockerfile `ENV` for defaults, env file for overrides.

2. **macOS Docker cannot `chown` bind-mounted files to a different UID.**
   The virtiofs/grpcfuse driver silently blocks ownership changes. `chown`
   to the same UID succeeds (no-op), but `chown` to a different UID fails
   with "Permission denied" — even as root. This means the PUID/PGID must
   exactly match the host user for macOS bind mounts to work. The init-
   usermap chown is effectively a no-op on macOS when configured correctly,
   and fatally fails when misconfigured (which was the symptom that led to
   discovering the env_file override bug).

3. **`with-contenv` is required for s6-rc services to see env vars.**
   s6-overlay does not automatically pass the container's environment to
   services. Without `with-contenv`, PUID/PGID were invisible and always
   defaulted to 1000.

## Open questions / future work

- **obsidian-headless config persistence**: Consider setting
  `XDG_CONFIG_HOME=/config` so ob's cached sync state survives container
  recreations (would avoid full re-syncs on restart). Needs investigation
  of what ob stores beyond the auth token.

- **Collapse s6-overlay Dockerfile layers**: The noarch tarball, arch
  tarball, and cleanup are three separate layers. Collapsing them would
  save ~6 MB of intermediate layer storage. Low priority since the final
  image size is unaffected in single-platform builds.

- **git-watcher SIGTERM trap**: The trap in the pipe subshell is dead code
  (traps don't fire in pipe subshells). The finish script handles shutdown
  commits correctly, but rewriting with a named pipe would be cleaner.

- **git remote write-only warning**: Document that users should not manually
  push to the backup remote, as `git pull --rebase` could introduce changes
  that Obsidian Sync then uploads to the cloud.

- **LLM commit messages**: Still planned, not implemented.
