# Session 006 — Add log prefix to obsidian-sync output

**Date:** 2026-03-02

## Summary

Added `[obsidian-sync]` prefix to all output from the `ob sync --continuous`
process, making it distinguishable from `[git-watcher]` output in
`docker logs`.

## Problem

The obsidian-headless binary prints bare messages like "Fully synced" to
stdout. These appear in `docker logs` without any indication of which
service produced them, making interleaved logs hard to read.

## What was done

1. **Initial approach**: piped `ob` output through `sed -u` to add a prefix.
2. **Subagent reviews** identified three critical issues:
   - s6 SIGTERM doesn't reach `ob` through a pipeline (orphaned processes)
   - `sed` exit 0 masks `ob`'s real exit code from s6
   - SIGPIPE risk if `sed` dies before `ob`
3. **BusyBox testing** revealed `sed -u` is not supported on Alpine.
4. **Final solution**: `log-prefix.sh` — a FIFO-based wrapper that:
   - Creates a named FIFO for output capture
   - Runs the command as a direct child (proper signal delivery)
   - Uses `while read` + `printf` instead of `sed -u`
   - Captures and returns the real exit code
   - Forwards SIGTERM/SIGINT to the child process

## Also fixed

- `.gitignore`: `local/` pattern was matching `rootfs/usr/local/` — anchored
  to repo root with `/local/`.

## Decisions

- **ADR-0004**: Documented the FIFO-based log prefixing decision.
- Deferred heartbeat deduplication ("Fully synced" noise) to a future session.

## Open questions

- Should "Fully synced" heartbeats be deduplicated or suppressed?
  (UX reviewer says yes; deferred for now to keep wrapper simple.)
- The obsidian-expert raised a valid concern about partial sync commits:
  if Obsidian downloads 50 files and the debounce window is too short,
  git could commit a partial sync state. Consider parsing "Fully synced"
  as a sync-complete signal in the future.
