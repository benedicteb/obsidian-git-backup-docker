# Session 001 — Initial PoC Implementation

**Date:** 2026-03-01

## What was accomplished

Built the complete initial PoC for obsidian-git-backup-docker from scratch.
Starting from only a spec and agent scaffolding, the session produced a fully
structured Docker image ready for testing.

### Files created

- `Dockerfile` — node:22-alpine + s6-overlay v3, obsidian-headless, git,
  inotify-tools
- `docker-compose.yml` — Example compose with documented volumes
- `.gitignore` — Project-level ignores
- `.env.example` — Documented env vars with security warnings
- `README.md` — Quick start, architecture, env var reference, limitations
- `rootfs/usr/local/bin/init-config.sh` — Oneshot init: env validation, SSH
  key generation/setup, git clone with retry, vault .gitignore, ob sync-setup
- `rootfs/usr/local/bin/git-watcher.sh` — Longrun watcher: inotifywait with
  debounce, git commit/push, pull --rebase before push, graceful shutdown
- `rootfs/etc/s6-overlay/s6-rc.d/` — Three s6-rc service definitions
  (init-config oneshot, obsidian-sync longrun, git-watcher longrun + finish)
- `docs/adr/0001-base-image-and-s6-overlay.md` — ADR for base image choice
- `docs/poc_plan.md` — Full implementation plan with decisions

## Decisions made

| Decision | Choice |
|---|---|
| Base image | `node:22-alpine` + s6-overlay v3.2.2.0 |
| Process supervision | s6-overlay with s6-rc service definitions |
| Git auth | SSH only, auto-generate key if missing, retry loop |
| SSH known_hosts | `StrictHostKeyChecking=accept-new` (TOFU) |
| Git repo init | Clone existing remote on startup |
| First-start order | Git clone first, then obsidian-headless syncs on top |
| Crash behavior | s6 auto-restarts both longrun services |
| Obsidian auth | `OBSIDIAN_AUTH_TOKEN` env var (read by ob directly) |
| Volumes | `/config` required (SSH keys), `/vault` optional |
| LLM commit messages | Deferred to future iteration |
| Watcher | inotifywait + debounce timer |
